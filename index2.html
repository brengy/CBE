Image Format Requirements

  • U1 Files: Front side of cheque (U1.tiff or U1.tif)
  • F1 Files: Back side with MICR line (F1.tiff or F1.tif)
  • Both files must have matching base names (e.g., CHQ001_U1.tiff and CHQ001_F1.tiff)
  • Supported formats: .tiff, .tif, .png, .jpg, .jpeg, .bmp

üöÄ Operation Steps

1 Configure Processing Options

Before starting, configure the checkboxes in the UI to control what features are enabled:

‚úì Pattern Detection (pattern Checkbox)

Enables ROI pattern analysis. When disabled, all pattern checks are skipped.

‚úì Quick Scan Mode (Micr_Ink_Detection_Only Checkbox)

Fast processing mode - only checks MICR ROI (2nd pattern) with 55% threshold. Automatically enables Pattern Detection and disables Version_detection, Fraudulent, Bad Print, and Visualization.

‚úì Version Detection (versionCheckbox)

Detects if cheque is "Old" or "New" based on oseparator and character analysis. Affects ROI checking logic (Old cheques only check MICR pattern).

‚úì Bank Name Mode (Bank Name Checkbox)

When CHECKED: Uses default bank name from textbox, skips all F1 file processing. When UNCHECKED: Extracts bank from F1 MICR Bank Code.

‚úì Fraudulent Detection (fraudulent Checkbox)

Analyzes character whiteness in currency ROI. Marks cheques with >78% white pixels as fraudulent (listBox11).

‚úì Bad Print Detection (bad Print Checkbox)

Detects printing issues (no currency found, out of bounds, etc.). Results go to listBox13.

‚úì Visualization (visualization Checkbox)

Shows debug images with currency location, ROI, and search regions. Only displays for items in fraudulent or bad print lists.

2 Select Input Folder (Optional)

Click "Browse Folder" button to select a custom input folder. If not selected, uses input_cheques_folder from config.json.

3 Set Default Bank Name (if using Bank Name Mode)

If bank Name Checkbox is checked, enter or select bank name in textbox/dropdown. This will be used for all cheques.

4 Start Processing

Click "Start button" to begin processing. Progress bar shows completion percentage (0-100%).

5 Review Results

Check the various listboxes to review categorized results. Click any item to view corresponding U1/F1 images in picture boxes.

‚öôÔ∏è Processing Phases

Phase 1: File Processing & Classification (0-70%)

  1. F1 File Handling:
    • If Bank Name Mode is ON: Skip F1 processing, use default bank
    • If Bank Code Mode is ON: Process F1 for MICR extraction
    • Missing F1 ‚Üí Uses default bank name, continues processing
  2. Bank Code Extraction (if enabled):
    • Locate MICR separator in F1 image
    • Extract 4-digit serial after separator
    • Map serial to bank name using bank_mapping_file
    • Apply 3 vs 8 correction algorithm
  3. Version Detection (if enabled):
    • Search for oseparator in bottom right of F1
    • Progressive search: very bottom ‚Üí bottom quarter ‚Üí bottom third
    • Analyze character size after oseparator
    • Classify as "New" or "Old"
  4. ROI Pattern Analysis (if enabled):
    • Quick Scan: Only check MICR ROI (55% threshold)
    • Normal Mode: Check all ROIs (70% threshold)
    • Old cheques: Only check MICR pattern
    • New cheques: Check all patterns
  5. Currency & Character Analysis (if enabled):
    • Search for currency symbol in upper 2/3, right half
    • Extract ROI based on currency location + offsets
    • Segment white characters using connected components
    • Calculate white pixel percentage per character
    • Mark as fraudulent if >78% white pixels

Phase 2: Logo Detection & Finalization (70-100%)

  1. Logo Detection (if enabled):
    • Full Mode: Check all cheques
    • Selective Mode: Only check specific categories
    • Filter logos by detected bank name
    • Three-tier classification:
      • >0.7: High confidence (Success)
      • 0.55-0.7: Medium confidence (Lacks - RecheckL)
      • <0.55: Low confidence (Failed - RecheckF)
  2. Special Bank Identification:
    • Check if bank name exists in exbanks.txt
    • Add to listBoxEx if special bank detected
  3. Final Result Generation:
    • Combine all processing results
    • Assign FinalResult status
    • Populate listBoxFinal with all recheck items
    • Export to CSV (cheque_processing_results.csv)

‚öñÔ∏è Processing Conditions & Thresholds

Template Matching (config.json)

  • separator_score: 0.5 (MICR separator detection)
  • currency_score: 0.5 (Currency symbol matching)
  • oseperator_score: 0.5 (Old cheque separator)

Logo Detection Thresholds

  • high_confidence: 0.7 (Success - listBoxLogoSuccess)
  • medium_confidence: 0.55 (Lacks - listBoxLogoLacks, RecheckL)
  • low_confidence: 0.55 (Failed - listBox1, RecheckF)

ROI Analysis Thresholds

  • normal_mode: 70% BPR (Black Pixel Ratio)
  • quick_scan_mode: 55% BPR (MICR pattern only)

Fraudulent Detection

  • white_pixel_percentage: 78.0% (Character tampering threshold)

Character Segmentation

  • min_width: 3 pixels
  • max_width: 25 pixels (250 for currency analysis)
  • min_height: 15 pixels

Version Detection Logic

New Cheque: Character width >3 AND height >5 after oseparator
Old Cheque: Smaller characters or no characters found

‚ö†Ô∏è Important Warnings

CSV File Lock: Close Excel or any program using cheque_processing_results.csv before processing. The application checks for file locks and will warn you if the file is in use.
Bank Name Mode Impact: When bank Code Checkbox is ON, ALL F1 processing is skipped. This means:
  • Version detection always returns "New"
  • No MICR serial extraction
  • No orphaned F1 detection
  • All cheques use the default bank name
Missing F1 Files: When Bank Code mode is ON and F1 is missing, the system now:
  • Uses default bank name instead of stopping
  • Continues processing the U1 file
  • Adds to listBox3 but doesn't skip the cheque
Pattern Detection Requirement: Quick Scan mode requires Pattern Detection to be enabled. The checkbox automatically enables it.
Visualization Display: Debug visualizations only appear for cheques in listBox11 (fraudulent) or listBox13 (bad print), even when visualization is enabled.

‚úÖ Best Practices

For Fast Processing (Batch Operations)

  1. Enable Quick Scan Mode
  2. Enable Bank Name Mode if all cheques are from same bank
  3. Disable Visualization, Fraudulent, and Bad Print checks
  4. Review listBox5 and listBox8 for pattern results

For Complete Analysis (Quality Control)

  1. Disable Quick Scan
  2. Enable Version, Fraudulent, Bad Print detection
  3. Enable Logo Detection (Full Mode)
  4. Enable Visualization for problematic cases
  5. Review all listboxes systematically

For Mixed Bank Processing

  1. Disable Bank Name Mode (use Bank Code detection)
  2. Ensure bank_mapping_file is up to date
  3. Enable Logo Detection with Selective Mode
  4. Check listBoxEx for special banks

üìÑ CSV Output Format

The application exports results to cheque_processing_results.csv with the following columns:

Filename | LogoResult | LogoScore | SerialMatches | SerialString | BankName | OcrText | ExtractedDigits | FinalResult | RoiOverall | ChequeVersion | Fraudulent | IsSpecialBank
  • FinalResult: Bank name, "RecheckF" (failed), or "RecheckL" (lacks confidence)
  • RoiOverall: "RO Accepted", "RO Recheck", "RO Skipped", or "Pattern Detection Disabled"
  • ChequeVersion: "New", "Old", "Unknown", or specific error message
  • Fraudulent: "Yes" or "No"
